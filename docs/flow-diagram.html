SoulSeed Compass + HOLO Core

What we've built, in pictures · v0.01 (live) · 2026-06-05

The core idea: one engine, swappable skins. The persona (REZZIE / COACH) lives in a data manifest — the Skin — never inside the engine — the Core. A second product is a new manifest on the same Core, with zero engine code (proven by an automated "seam test"). Three views: the architecture, one live agent turn, and the journey.

1 · The three layers — Core / Skin / Instance

The engine knows nothing about REZZIE. Swap the manifest → different product; the "HOLO CORE" boxes never change.

2 · One agent turn, end to end (live today)

A user answers in a chamber and REZZIE replies — including the new safety nets (auto-retry + graceful fallback) so a model blip never breaks the journey.

sequenceDiagram
  autonumber
  actor U as User
  participant UI as SoulSeed UI
  participant SDK as @holo/sdk
  participant API as /v1/agents/run
  participant Core as runAgent (Core)
  participant Skin as Manifest (Skin)
  participant Mem as Memory Lattice
  participant LLM as Model Router
  participant DB as Supabase
  U->>UI: types an answer
  UI->>SDK: run(chamber, input)
  SDK->>API: POST /v1/agents/run
  API->>API: validate (Zod) + authorize
  API->>Core: delegate (no logic in the route)
  Core->>Skin: getAgent(manifest, "rezzie")
  Note over Core,Skin: voice = SKIN, shape = ENGINE
  Core->>Mem: read memory context (cross-product)
  Core->>LLM: generate(voice + engine JSON contract)
  LLM-->>Core: structured output (retry once, else safe fallback)
  Core->>DB: persist agent_run + memory writes
  Core-->>API: SoulSeedAgentOutput
  API-->>SDK: JSON
  SDK-->>UI: REZZIE's reply
  UI-->>U: renders the message
